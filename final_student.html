This project leverages the Online News Popularity dataset to develop machine learning models that predict the popularity of news articles published on Mashable, a leading online news platform. The dataset contains 39,644 articles described by 61 features, including content-related attributes (word counts, keywords, sentiment), publishing metadata (day of week, data channel), and the number of social media shares each article received.

Some features within the dataset may require further elaboration:

  • kw_min_min: The minimum number of shares among articles with the least popular keyword in the metadata. For example, if an article has keywords A (min shares = 10), B (min shares = 20), and C (min shares = 15), then kw_min_min = 10.
  • self_reference_min_shares: The minimum number of shares among all Mashable articles referenced within the current article.
  • LDA features (LDA_00 through LDA_04): Probabilities of the article belonging to each of five precomputed topics derived using Latent Dirichlet Allocation. These sum to 1 for each article.
  • Sentiment features: global_sentiment_polarity (overall sentiment, −1 to 1), global_rate_positive_words, global_rate_negative_words, avg_positive_polarity, min_positive_polarity, max_positive_polarity, avg_negative_polarity, min_negative_polarity, max_negative_polarity.

Bonus: Stacked Ensemble (15 bonus points)

Stacking (also called stacked generalization) is an ensemble technique that combines the predictions of multiple diverse models using a meta-learner. The idea is that different models capture different patterns in the data, and a meta-learner can learn the optimal way to combine their predictions — often outperforming any individual model.

How Stacking Works

  1. Train base models: You already have three base models — Random Forest, GBM, and XGBoost.
  2. Generate out-of-fold predictions: To avoid data leakage, base model predictions for the meta-learner must be generated on data the base model did not train on. Use k-fold cross-validation:
    • Split training data into k folds
    • For each fold, train each base model on the other k−1 folds and predict on the held-out fold
    • After all folds, you have OOF (out-of-fold) predictions from each base model for every training observation
  3. Train the meta-learner: Use the OOF predictions as new features (one column per base model) and the original target as the response. A simple model works best — linear regression (for regression) or logistic regression (for classification).
  4. Make final predictions: For the test set, predict with each base model (trained on full training data), then feed those predictions into the meta-learner.

Why Stacking Works

Each base model makes different errors on different observations. If RF overpredicts for certain types of articles while XGBoost underpredicts for those same articles, a meta-learner can learn to weight their predictions optimally. The key requirement is model diversity — stacking works best when base models are substantially different (which RF, GBM, and XGBoost are, due to their different learning mechanisms).
